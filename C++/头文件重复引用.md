# 头文件重复引用

在c和c++中，如果一个头文件被多个文件同时引用，如果不进行额外的处理，会出现"duplicate symbol"的错误。这是因为每一次引用头文件，头文件中所定义的方法和变量都会在这个文件中再次出现

所以一般都会添加下面的宏来防止重复定义

```c++
#ifndef TEST_H
#define TEST_H
...
#endif
```

或

```c++
#program once
```



> 在头文件被多处引用的情况下，方法和变量的声明可以在头文件中，方法和变量的声明**绝对**不能出现在头文件中
>
> 除非这个方法是模版类方法

变量的声明也不能出现在头文件中，除非这个变量是全局变量，下面的声明是可以出现在头文件中的

```
extern int test
```

方法和变量的定义都应该出现在源文件中
