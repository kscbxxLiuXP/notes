# commit备注

## 简介

在git commit 中填写的备注有一些统一的规范

对于描述提交所做的更改非常重要，良好的提交消息可以帮助团队更好地理解代码变更的目的和内容。以下是一些常见的提交消息的类型：

1. **feat**: 新增功能（feature）
2. **fix**: 修复 bug
3. **docs**: 文档变更
4. **style**: 代码格式、空格、分号等不影响代码逻辑的修改
5. **refactor**: 重构代码，既不修复 bug 也不增加新功能
6. **test**: 添加或修改测试代码
7. **chore**: 构建过程或辅助工具的修改，不涉及源码修改
8. **perf**: 优化性能的代码更改
9. **revert**: 撤销之前的提交
